PayPal Settings

You must set up the following settings to enable your PayPal shop:

General

Use PayPal

If disabled, the order will not be submitted to PayPal. Instead, only an email will be sent to the recipient specified in the Email settings.

Note: This feature requires PHP and server side database (Access/MySQL/SQLite), it cannot be used with JavaScript database type.

PayPal Account

Your registered email address (PayPal Account) in PayPal

Enter your Paypal account into the [PayPal account] textbox in the form of PayPal registered email. If you use Sandbox, enter your test account. If not, enter your real account.

 

Currency

Currency (currency_code) Defines the currency in which the monetary variables are denoted.
Currency symbol

The currency symbol or dollar sign before the amount.

Note: If the system encoding of your PC is utf-8 or windows-1252, you can use the euro currency symbol, €. Otherwise, you should not use it because the symbol is not supported by other character sets. If your system encoding is not utf-8 or windows-1252 but you have to use the symbol, you need to use utf-8 project, set the project charset as "utf-8" (see HTML Settings) and enable UTF8Output (see Generate Settings).
Decimal symbol The decimal symbol of the price.
Grouping symbol The thousand separator.
No. of digits after decimal The number of digits after decimal. In normal cases it should be 2.

 

Payment Data Transfer (PDT)

Payment Data Transfer (PDT) provides merchants with the ability to display transaction details to buyers who are redirected back to their site upon payment completion.

If you want to enable this feature, check [Use Payment Data Transfer (PDT)] and paste your identity token from PayPal.com to the [Identity Token] textbox.

Notes: You must login PayPal.com and enable Auto Return for Website Payments to use Payment Data Transfer. When you save your PDT preferences at PayPal.com, a message appears at the top of the page indicating that you have successfully saved your preferences. Your identity token also appears in this message. You will need to copy this identity token to PayPal Shop Maker. Also see Payment Data Transfer (PDT) for more details.

 

Instant Payment Notification (IPN)

Instant Payment Notification (IPN) allows you to integrate PayPal payments with your website's back-end operations. IPN provides immediate notification and confirmation of PayPal payments you receive.

If you want to enable this feature, check [Use Instant Payment Notification (IPN)].

Notes: You also need to setup IPN at your PayPal account, see Instant payment Notification (IPN) for more details.

 

Digital Goods

If you sell digital goods, you can allow customers instant access to the download URLs with an email (via IPN) or a return page (via PDT). To enable, check [Use digital goods download].

There are 2 types of download method. If the download files are not large in size, enable [Download via script], otherwise disable it. (See Digital Goods Download for explanation.)

To prevent fraudulent transactions, you can choose [Approval before providing download URL] to receive an email for your approval before sending the email to your customer. After the period of time specified in [Download timeout (hours)], the download URLs will become invalid.

Also see Digital Goods Download for more details.

 

Sandbox

PayPal provides a Sandbox test environment, you can use test accounts to try out the payment process. If you have a Sandbox test account, check [Use PayPal Sandbox]. If not, just left it unchecked now.

To test in the Sandbox, you must first create a Developer Central account at PayPal.com. With this account, you can create multiple PayPal accounts, such as seller and buyer accounts, to simulate different scenarios.

For complete details about using Developer Central and the Sandbox, see the Sandbox User Guide by PayPal.

Notes:

  1. After testing, remember to uncheck this box and generate production version of scripts.

  2. If you only uncheck this option to generate production version and make no other changes in the project, you don't need to re-upload all pages. You can upload ppcfg.php only.

 

Other variables

HTML Variables for Payment Transactions
address_override

If enabled, the address specified with automatic fill-in variables overrides the PayPal member’s stored address. Buyers see
the addresses that you pass in, but they cannot edit them. PayPal does not show addresses if they are invalid or
omitted.

 
weight_unit The unit of measure if weight_cart is specified. Possible values are:
lbs or kgs
Defaults to lbs.
HTML Variables for Shopping Carts
paymentaction Indicates whether transaction is an authorization to be captured later with PayPal Authorization & Capture on https://www.paypal.com/. For more information, see the PayPal Authorization & Capture Integration Guide.
handling_cart

Single handling fee to be charged cart-wide.

Note: If you use Custom shipping type, this fee will be added to the total of handling fee for individual items. Otherwise, it will be posted to PayPal directly.
HTML Variables for Displaying PayPal Checkout Pages
page_style

Sets the Custom Payment Page Style for payment pages associated with this button/link. Allowed values are:
somePageStyleName: the name of a page style you have defined
primary: always use the page style set in the account profile as primary.
paypal: use the PayPal page style.
Alphanumeric ASCII lower 7-bit characters only, plus underscore. No spaces. (Max 30 chars)

image_url The URL of the 150x50-pixel image you would like to use as your logo. If omitted, the customer sees your business name (if you have a Business account) or your email address (if you have Premier account). (Max 1,024 chars)
cpp_header_ image Sets the image you choose to appear at the top left of the payment page. The image can have a maximum size of 750 pixels wide by 90 pixels high. Valid character set is URL-safe characters. The cpp_header_image variable has no character limit because the image is not stored on our database. It is recommended that you only provide an image that is stored on a secure (https) server.
cpp_headerback_color
Sets the background color for the header of the payment page. Valid character set is 6 character HTML Hex code in ASCII. This variable is not case-sensitive.
cpp_headerborder_color
Sets the border color around the header of the payment page. The border is a 2-pixel perimeter around the header space, which has a maximum size of 750 pixels wide by 90 pixels high. Valid character set is 6 character HTML Hex code in ASCII. This variable is not case-sensitive.
cpp_payflow_color
Sets the background color for the payment page below the header. Valid character set is 6 character HTML Hex code in ASCII. This variable is not case sensitive. Certain background colors that conflict with PayPal's error messages are not allowed.
no_note Including a note with payment. If checked, your customer will not be prompted to include a note. Otherwise your customer will be prompted to include a note.
cn Label that will appear above the note field. This value is not saved and will not appear in any of your notifications. If omitted, no variable will be passed back to you. (Max 60 chars)
cs Sets the background color of your payment pages. If checked, the background color will be black. Otherwise the background color will be white.
no_shipping The buyer's shipping address.
Omitted or set to 0 = customer is prompted to include a shipping address
1 = customer is not asked for a shipping address.
2 = customer is required to provide a shipping address.
return The URL to which the buyer’s browser is redirected after completing the payment; for example, a URL on your site that displays a “Thank you for your payment” page.
Default: buyer is redirected to a PayPal web page. (Max 1,024 chars)
rm

"Return method": FORM METHOD used to send data to the URL specified by the return variable after payment completion.
Allowed values are:
0: (default) all shopping cart transactions use the GET method
1: the buyer’s browser is redirected to the return URL by the GET method, and no transaction variables are sent
2: The buyer’s browser is redirected to the return URL by the POST method, and all transaction variables are also
NOTE: The rm variable takes effect only if the return variable is also set.
cbt Sets the text for the Continue button on the PayPal Payment Complete page. Requires Return URL to be set. (Max 60 chars)
cancel_return An internet URL where the user will be returned if payment is cancelled - for example, a URL on your site which hosts a "Payment Cancelled" page. If omitted, users will be taken to the PayPal website. (Max 1,024 chars)
